# Firmware Architecture

Single product firmware: multi-band Si4732 receiver with the Waveshare
2.8" ST7789V TFT touch shield. Drives the chip, reads the rotary
encoder, owns the UI, persists user state. The legacy 128Ă—64 SSD1315
OLED variant was retired so this doc covers only the TFT path.

| Env           | Entry point         | Purpose                               |
|---------------|---------------------|---------------------------------------|
| `esp32dev`    | `src/main.cpp`      | Shipping firmware                     |
| `shield_test` | `src/test_shield.cpp` | TFT shield hardware bring-up fixture |

`shield_test` is not a product and is not published by the release
workflow. See [display_shield_test.md](display_shield_test.md).

## Module map

| File                                   | Role                                                                 |

## `radio.cpp` â€” multi-band Si4732 layer

Public API declared in [include/radio.h](../include/radio.h):

| Function                                 | Purpose                                                         |
|------------------------------------------|-----------------------------------------------------------------|
| `radioInit()`                            | RC-reset-safe power-up, tune to the saved (or index-0) band     |
| `radioSetBand(idx)`                      | Reconfigure Si4735 via `setFM` / `setAM`, re-apply volume       |
| `radioGetBandIdx()` / `radioGetCurrentBand()` | Current band index + descriptor                           |
| `radioSetFrequency(f)`                   | Tune; band-clamped; clears RDS mirrors on FM                    |
| `radioGetFrequency()`                    | In the current band's native units                              |
| `radioFormatFrequency(buf, size)`        | Renders "102.4 MHz" or "1530 kHz" depending on band mode        |
| `radioSetVolume(v)` / `radioGetVolume()` | Clamped to `MAX_VOLUME`                                         |
| `radioPollSignal()`                      | Self-rate-limited 500 ms; caches RSSI / SNR / stereo pilot      |
| `radioGetRssi()` / `radioGetSnr()` / `radioIsStereo()` | Cached values; stereo forced false off-FM         |
| `radioPollRds()`                         | Self-rate-limited 200 ms; no-op on AM bands; diffed mirror      |
| `radioGetRdsPs()` / `radioGetRdsRt()`    | Empty on AM or after 10 s sync loss                             |

Poll functions return `true` when a cached value changed, letting the UI
layer drive its dirty flags without peeking at library internals.
`radio.cpp` is not thread-safe; the main loop is the sole caller. The
Berndt-inspired dual-core task split will add proper synchronization
when it lands.

## `input.cpp` â€” encoder layer

Pins fixed at A=18, B=19, BTN=5. Public API ([include/input.h](../include/input.h)):

| Function                                   | Purpose                                                           |
|--------------------------------------------|-------------------------------------------------------------------|
| `encoderInit()`                            | Pins + ISR + acceleration + state reset                           |
| `encoderSetBoundsForMode(mode, freq, vol)` | Bounds derived from `radioGetCurrentBand()` in FREQUENCY mode     |
| `encoderSetBoundsForMenu()`                | Wide no-wrap range; callers read deltas                           |
| `encoderPollRotation(out&)`                | True on change; fills raw value                                   |
| `encoderPollButton()`                      | Returns `BTN_NONE` / `BTN_CLICK` / `BTN_LONG_PRESS` (500 ms)      |
| `enum AdjustMode { MODE_FREQUENCY, MODE_VOLUME };` |                                                           |
| `enum ButtonEvent { BTN_NONE, BTN_CLICK, BTN_LONG_PRESS };` |                                                |

Long-press discrimination: the underlying AiEsp32RotaryEncoder library
only exposes click-on-release, so `encoderPollButton()` runs its own
state machine on top of `isEncoderButtonDown()` to distinguish the two
events. Exactly one `BTN_LONG_PRESS` fires per hold; the subsequent
release is silent so long presses never emit a stray `BTN_CLICK`.

## `menu.cpp` â€” modal menu

Two states today:

- `MENU_TOP` â†’ items `Band` and `Close`.
- `MENU_BAND` â†’ one row per band in `g_bands[]` plus `< Back`.

Public API ([include/menu.h](../include/menu.h)): `menuIsOpen()`,
`menuOpen()`, `menuClose()`, `menuHandleRotation(delta)`,
`menuHandleClick()`, `menuTakeDirty()`, `menuDraw(tft)`.

The menu is a full-screen takeover: while open, `main.cpp` skips the
zone-based dirty-flag pipeline and only calls `menuDraw()` when
`menuTakeDirty()` returns true. On close, `main.cpp` forces `DIRTY_ALL`
to repaint the main UI from scratch. ATS-Mini's `CMD_*` namespace
(`CMD_BAND = 0x1000`, etc. â€” see their `ats-mini/Menu.h`) is adopted so
follow-up PRs can add `CMD_VOLUME`, `CMD_STEP`, `CMD_MODE`, `CMD_MEMORY`
without renaming.

## `persist.cpp` â€” versioned NVS

Namespace `"radio"`, schema version `PERSIST_SCHEMA_VER`. On mismatch
the namespace is wiped and defaults re-applied.

| Key         | Type | Meaning                                                |
|-------------|------|--------------------------------------------------------|
| `ver`       | u16  | `PERSIST_SCHEMA_VER`                                   |
| `band`      | u8   | Current band index                                     |
| `vol`       | u8   | Global volume 0..`MAX_VOLUME`                          |
| `freq<N>`   | u16  | Per-band last-tuned frequency in that band's native units (up to 16 bands) |

Writes coalesce with a â‰Ą1 s per-key rate limit so rapid encoder rotation
doesn't hammer flash. `persistFlush()` forces any pending writes out
(intended for a future sleep path).

## `main.cpp` â€” top-level

`setup()` order: Serial â†’ `Wire.begin` â†’ `initBacklight()` â†’ `initDisplay()`
â†’ `initGauge()` (needle sprite) â†’ `tft.setTouch(TOUCH_CALIBRATION)` â†’
splash â†’ 500 ms RC-reset wait â†’ `persistInit()` (load saved state) â†’
`radioInit()` â†’ `radioSetBand(saved)` â†’ `encoderInit()` â†’
`encoderSetBoundsForMode(...)` â†’ first full paint.

`loop()` routes input to the menu when `menuIsOpen()`, otherwise to the
frequency / volume / mode handlers. Radio polling + needle animation +
dirty-flag redraw are suppressed while the menu is open â€” they resume
on close.

## Si4732 init sequence (RC-reset-safe)

1. `setDeviceI2CAddress(0)` â€” IÂ˛C address 0x11 (SEN = GND).
2. `setPowerUp(...)` â€” FM mode, crystal oscillator, analog audio.
3. `radioPowerUp()` â€” send POWER_UP over IÂ˛C (chip is already out of
   hard reset thanks to the external RC circuit; no GPIO toggle
   required).
4. `delay(250)` â€” crystal oscillator stabilisation.
5. `setVolume(persisted)`, `getFirmware()`.
6. `applyBandToRadio()` â€” `setFM(...)` / `setAM(...)` using the current
   band's table entry; on FM also enables RDS via
   `setRdsConfig(1, 3, 3, 3, 3)`.

After step 3 all other PU2CLR methods work normally over IÂ˛C; no
`reset()` is ever called.
