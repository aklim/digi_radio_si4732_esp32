## `radio.cpp` â€” shared Si4732 layer

Public API declared in `include/radio.h` ([link](../include/radio.h)):

| Function                | Purpose                                                    |
|-------------------------|------------------------------------------------------------|
| `radioInit()`           | RC-reset-safe power-up, tune 102.4 MHz, enable RDS         |
| `radioSetFrequency(f)`  | Tune; clear local RDS mirrors so old text does not linger  |
| `radioGetFrequency()`   |                                                            |
| `radioSetVolume(v)` / `radioGetVolume()` |                                           |
| `radioPollSignal()`     | Self-rate-limited 500 ms; caches RSSI / SNR / stereo pilot |
| `radioGetRssi()` / `radioGetSnr()` / `radioIsStereo()` |                             |
| `radioPollRds()`        | Self-rate-limited 200 ms; copies library PS/RT into local mirrors, clears after 10 s no-sync |
| `radioGetRdsPs()` / `radioGetRdsRt()` |                                              |

The poll functions return `true` when a cached value changed, letting the UI
layer drive its dirty flags without peeking at library internals. `radio.cpp`
is not thread-safe; both mains call it from the Arduino loop task.

## `input.cpp` â€” shared encoder layer

Pins fixed at A=18, B=19, BTN=5. Public API ([include/input.h](../include/input.h)):

| Function                                   | Purpose                          |
|--------------------------------------------|----------------------------------|
| `encoderInit()`                            | Pins + ISR + acceleration        |
| `encoderSetBoundsForMode(mode, freq, vol)` | Reconfigure bounds / wrap       |
| `encoderPollRotation(out&)`                | True on change; fills raw value  |
| `encoderPollButton()`                      | True when the button was clicked |
| `enum AdjustMode { MODE_FREQUENCY, MODE_VOLUME };` |                          |

Touch (XPT2046) lives in `main_tft.cpp` so the OLED build does not drag in
`TFT_eSPI.h` for a helper it would never call.

## OLED entry â€” `src/main.cpp`

`setup()` order: Serial â†’ Wire.begin â†’ 500 ms RC-reset wait â†’ `initDisplay()`
â†’ `radioInit()` â†’ `encoderInit()` â†’ `encoderSetBoundsForMode(...)`.

`loop()` is four calls: `encoderPollRotation`, `encoderPollButton`,
`radioPollSignal`, `updateDisplay`. The display uses a single
`displayNeedsUpdate` boolean â†’ full frame buffer + one flush per change.

OLED UI layout:

```
+------------------------------+
| FM          [===RSSI===]     |  y = 0..15   (yellow)
|------------------------------|
| > 102.4 MHz                  |  y = 22..38  (blue)
|                              |
| > Vol [========--] 30        |  y = 52..60  (blue)
+------------------------------+
```

The `>` marker appears next to the parameter currently controlled by the
encoder. `FW_VERSION` is embedded in the binary but intentionally not
rendered on the OLED (see [../CLAUDE.md](../CLAUDE.md)).

## TFT entry â€” `src/main_tft.cpp`

`setup()` adds `initBacklight()` (LEDC PWM on GPIO 4) and
`tft.setTouch(TOUCH_CALIBRATION)` before the splash, then mirrors the OLED
sequence.

`loop()` adds `radioPollRds()` and `handleTouch()` to the rotation cycle.
Touch taps force focus to the matching zone (FREQ or VOL) with a 200 ms
debounce and a 15 ms settle delay; the encoder continues to drive value
changes.

UI uses a per-zone dirty-flags byte:
`DIRTY_HEADER | DIRTY_FREQ | DIRTY_RDS | DIRTY_METER | DIRTY_VOL | DIRTY_FOOTER`.
Full layout specification, coordinates, and touch zones live in
[display_tft.md](display_tft.md).

## Si4732 init sequence (RC-reset-safe)

Both variants use the same init path via `radioInit()`:

1. `setDeviceI2CAddress(0)` â€” IÂ²C address 0x11 (SEN = GND).
2. `setPowerUp(...)` â€” FM mode, crystal oscillator, analog audio.
3. `radioPowerUp()` â€” send POWER_UP over IÂ²C (chip is already out of hard
   reset thanks to the external RC circuit; no GPIO toggle required).
4. `delay(250)` â€” crystal oscillator stabilisation.
5. `setVolume(DEFAULT_VOLUME)`, `getFirmware()`.
6. `setFM(8700, 10800, 10240, 10)` â€” band limits and default 102.4 MHz.
7. `setRdsConfig(1, 3, 3, 3, 3)` â€” enable RDS with permissive error thresholds.

After step 3 all other PU2CLR methods work normally â€” they communicate over
IÂ²C and never call `reset()` again.
